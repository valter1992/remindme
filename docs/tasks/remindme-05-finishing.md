# RemindMe — Этап 5: Завершение

## Current State

После этапа 4 вся функциональность MVP реализована (напоминания с надёжной доставкой, заметки, задачи с inline-управлением). Не хватает полного покрытия тестами, end-to-end smoke-сценария, контейнеризации и документации запуска.

## Description

Стабилизировать MVP: закрыть пробелы в тестах по стратегии ТЗ, добавить end-to-end smoke, контейнеризацию (Docker Compose), документацию запуска и проверить все критерии приёмки.

## Scope

**In scope:**
- Полное покрытие тестами по стратегии ТЗ:
  - unit — парсер, работа со временем, formatter, сервисы сущностей (закрытие пробелов после этапов 1–4);
  - integration — SQLAlchemy-репозитории на временной SQLite, миграции, блокировка и повторные попытки worker;
  - bot handlers — маршрутизация команд и текста, callback-действия, личные и групповые чаты с mock Telegram API.
- End-to-end smoke: создание короткого напоминания тестовым ботом и получение уведомления после перезапуска приложения (единственный тест с реальным Telegram-клиентом, опциональный/ручной запуск).
- `Dockerfile` и `docker-compose.yml` (сервис бота + том под `data/`, `.env`); точка входа применяет миграции (`alembic upgrade head`) перед запуском бота.
- Документация запуска локально и через Docker Compose.
- Проверка всех 11 критериев приёмки ТЗ по чек-листу.
- Финальная чистка: `ruff check`, `ruff format`, `pytest tests/ -x`; `.gitignore` покрывает `data/`, `.env`, `.venv`.

**Out of scope:**
- Любая новая функциональность — только стабилизация, тесты, упаковка, докеризация, документация.

## Acceptance Criteria

- Все 11 критериев приёмки ТЗ проходят (отдельный чек-лист по каждому).
- Нефункциональные требования выполнены: один процесс/экземпляр; все операции async; доставка ≤ 10 секунд после заданного времени при доступных Telegram и БД; unit покрывают парсер/время/formatter/сервисы; integration покрывают репозитории и worker.
- `docker compose up` поднимает бота, применяет миграции и отвечает на `/start`.
- После перезапуска контейнера сохранённые записи доступны; просроченное напоминание доставляется.
- `pytest tests/ -x`, `ruff check src/`, `ruff format --check` зелёные; в логах и ошибках нет токена и текстов записей.

## Stack

- **Infrastructure:** SQLite (WAL, `PRAGMA foreign_keys=ON`), Docker Compose

## Risks and Constraints

- End-to-end smoke требует тестового бота и токена; запускается вручную/опционально через `pytest.mark.skipif`.
- Нагрузочное требование «≥ 20 одновременно активных пользователей» обосновывается однопоточностью MVP и покрытием тестами изоляции, а не полноценной нагрузкой.
- Токен `TELEGRAM_BOT_TOKEN` и `.env` прокидываются в контейнер через docker-compose `environment`/`env_file`, не копируются в образ (`COPY .env` запрещён) — соответствует требованиям безопасности ТЗ.
