# aiosqlite — асинхронный драйвер SQLite

Домен: асинхронный доступ к файлу базы данных RemindMe.
Аудитория: конфигурация подключения в `src/db/session.py`.

aiosqlite используется как диалект-драйвер для SQLAlchemy 2.x async. Прямое обращение к API aiosqlite в проекте не требуется — вся работа с данными идёт через асинхронные сессии SQLAlchemy.

## URL подключения

Драйвер указывается в `DATABASE_URL` префиксом `sqlite+aiosqlite`.

```python
DATABASE_URL = "sqlite+aiosqlite:///data/remindme.db"
```

Путь по умолчанию — `data/remindme.db`. Директория `data/` должна существовать к моменту первого подключения; при локальном запуске она создаётся, а сама база `.gitignore`-ится и не публикуется.

## Роль в стеке

aiosqlite реализует PEP 249 поверх `sqlite3` в отдельном потоке, поэтому блокирующие файловые операции не останавливают event loop бота и worker уведомлений.

## PRAGMA на соединении

Внешние ключи и режим журнала `WAL` включаются для движка SQLAlchemy; конкретный listener и `create_async_engine` описаны в `sqlalchemy`.

## Тесты

Во встроенных тестах используется временный файл SQLite через `tmp_path`; URL формируется как `sqlite+aiosqlite:///<tmp>/test.db`, чтобы не затрагивать `data/remindme.db`.

## Ограничения

- SQLite по файлу не поддерживает параллельные писатели; MVP работает в одном экземпляре процесса, поэтому конкуренции за запись нет.
- Большие двоичные данные в MVP не хранятся; типы полей — текст и целые числа.
